## 1. 아이디어 개요

PPT에 넣을 내용:

- 프로젝트명: SpeakLog
- 말하면서 학습하고, 요약하며 기록하는 웹앱
- 학습 주제 입력 -> 청중 선택 -> 설명하기 -> 핵심 요약 -> 기록 저장

발표 대본:

안녕하세요. 제가 만든 프로젝트는 **SpeakLog**입니다.

SpeakLog는 사용자가 오늘 배운 내용을 직접 말로 설명하고, 설명이 끝난 뒤 핵심 내용을 요약해서 기록으로 남길 수 있는 학습 웹앱입니다.

보통 공부를 할 때 그냥 읽거나 듣기만 하면 내가 정말 이해했는지 확인하기 어렵습니다. 그래서 저는 "누군가에게 설명해보는 방식"을 앱으로 만들면 학습 내용을 더 잘 기억할 수 있을 것이라고 생각했습니다.

이 앱의 기본 흐름은 간단합니다. 먼저 학습 주제를 입력하고, 설명을 들어줄 청중을 선택합니다. 그다음 화면 속 청중에게 설명하듯이 말하고, 설명이 끝나면 핵심 내용을 직접 정리합니다. 마지막으로 기록을 저장하면 나중에 다시 확인할 수 있습니다.

## 2. 프로젝트 제작 동기

PPT에 넣을 내용:

- 설명하면서 공부하면 이해 부족한 부분을 알 수 있음
- 말하기 + 요약하기 + 기록하기를 하나로 연결
- 학습 습관을 만들기 위한 웹앱

발표 대본:

제가 이 프로젝트를 만들게 된 이유는, 공부할 때 단순히 외우는 것보다 **내 말로 설명해보는 과정**이 중요하다고 생각했기 때문입니다.

어떤 내용을 배웠을 때 머릿속으로는 아는 것 같지만, 실제로 말로 설명하려고 하면 막히는 부분이 생깁니다. 그 부분이 바로 내가 아직 정확히 이해하지 못한 부분입니다.

그래서 SpeakLog는 사용자가 배운 내용을 말로 설명해보고, 이후에 기억나는 내용을 직접 요약하면서 학습을 정리할 수 있도록 만들었습니다.

즉, 이 프로젝트의 목표는 단순한 메모장이 아니라, **말하기와 정리를 통해 학습을 내 것으로 만드는 도구**를 만드는 것이었습니다.

## 3. 주요 화면

PPT에 넣을 내용:

- 홈 화면
- 학습 설정 화면
- 설명 진행 화면
- 요약 작성 화면
- 기록 목록 화면
- 기록 상세 화면

발표 대본:

이제 주요 화면을 설명하겠습니다.

첫 번째는 **홈 화면**입니다. 홈 화면에서는 앱의 소개 문구와 학습 시작 버튼이 있고, 최근 학습 기록 3개를 확인할 수 있습니다.

두 번째는 **학습 설정 화면**입니다. 여기서는 사용자가 학습 주제를 입력하고, 설명을 들어줄 청중을 선택합니다.

세 번째는 **설명 진행 화면**입니다. 선택한 청중 영상이 나오고, 상단에는 학습 주제와 타이머가 표시됩니다. 사용자는 이 화면을 보면서 실제로 설명하듯 말할 수 있습니다.

네 번째는 **요약 작성 화면**입니다. 설명을 마친 뒤, 사용자가 기억나는 핵심 내용을 직접 입력하는 화면입니다.

다섯 번째는 **기록 목록 화면**입니다. 지금까지 저장한 학습 기록을 최신순으로 볼 수 있습니다.

마지막은 **기록 상세 화면**입니다. 특정 기록을 클릭하면 학습 주제, 날짜, 설명 시간, 선택한 청중, 핵심 요약 내용을 자세히 확인할 수 있습니다.

## 4. 프로젝트 시연

PPT에 넣을 내용:

1. 학습 시작 클릭
2. 주제 입력
3. 청중 선택
4. 설명 시작
5. 타이머 확인
6. 설명 완료
7. 요약 작성
8. 기록 저장 및 확인

발표 대본:

이제 프로젝트를 직접 시연해보겠습니다.

먼저 홈 화면에서 **학습 시작하기** 버튼을 클릭합니다. 그러면 학습 설정 화면으로 이동합니다.

여기에서 예시로 학습 주제를 입력하겠습니다. 예를 들어 "React 상태 관리"라고 입력할 수 있습니다.

그다음 설명을 들어줄 청중을 선택합니다. 청중을 선택하면 선택된 카드가 표시되고, 설명 시작 버튼을 누르면 설명 화면으로 이동합니다.

설명 화면에서는 선택한 청중 영상이 나오고, 위쪽에 주제와 타이머가 표시됩니다. 타이머는 1초마다 증가하면서 사용자가 얼마나 오래 설명했는지 보여줍니다.

설명이 끝나면 **설명 완료** 버튼을 누릅니다. 그러면 요약 화면으로 이동하고, 여기에서 방금 설명한 내용을 핵심만 정리해서 작성합니다.

마지막으로 **작성 완료** 버튼을 누르면 기록이 저장됩니다. 저장된 기록은 홈 화면의 최근 기록이나 전체 기록 화면에서 다시 확인할 수 있습니다.

## 5. 프로젝트 주요 코드

PPT에 넣을 내용:

- `App.js`: 전체 상태와 화면 전환 관리
- `useState`: 주제, 청중, 시간, 요약, 기록 저장
- `useEffect`: 타이머와 localStorage 저장
- `AudienceVideo`: 청중 영상 출력
- `RecordGrid`: 기록 카드 목록 출력

발표 대본:

이제 프로젝트의 주요 코드를 설명하겠습니다.

가장 중심이 되는 파일은 `App.js`입니다. `App.js`에서는 현재 화면, 학습 주제, 선택한 청중, 설명 시간, 요약 내용, 저장된 기록을 상태로 관리합니다.

먼저 화면 전환은 `screen`이라는 state로 처리했습니다.

```js
const [screen, setScreen] = useState('home');
```

`screen` 값이 `home`이면 홈 화면을 보여주고, `setup`이면 설정 화면, `session`이면 설명 화면을 보여주는 방식입니다. 즉, 페이지 이동 기능을 따로 쓰지 않고 React의 상태값으로 화면을 전환했습니다.

다음으로 타이머 기능입니다.

설명 화면에 들어가면 `useEffect` 안에서 `setInterval`이 실행되고, 1초마다 `elapsed` 값이 증가합니다.

```js
setElapsed((value) => value + 1);
```

그리고 설명 화면을 벗어나면 `clearInterval`로 타이머를 멈춥니다. 이렇게 해서 설명 중일 때만 시간이 측정되도록 했습니다.

기록 저장 기능은 `saveRecord` 함수에서 처리했습니다. 사용자가 요약을 작성하고 저장하면, 날짜, 주제, 청중, 설명 시간, 요약 내용을 하나의 객체로 만들어 `records` 배열에 추가합니다.

그리고 이 기록은 `localStorage`에 저장됩니다. 그래서 새로고침을 해도 기록이 사라지지 않습니다.

청중 영상은 `AudienceVideo` 컴포넌트에서 처리했습니다. 설명 중에는 영상이 자동 재생되고, 기록 카드나 선택 화면에서는 미리보기처럼 표시됩니다.

기록 목록은 `RecordGrid` 컴포넌트를 사용했습니다. 홈 화면과 전체 기록 화면에서 같은 카드 디자인을 재사용하기 위해 따로 컴포넌트로 분리했습니다.

정리하면, 이 프로젝트는 React의 `useState`로 데이터를 관리하고, `useEffect`로 타이머와 저장 기능을 처리한 구조입니다.

## 6. 개선해야 할 점

PPT에 넣을 내용:

- 음성 인식 기능 추가
- AI 피드백 기능 추가
- 기록 수정/삭제 기능
- 로그인 및 서버 저장
- 학습 통계 기능

발표 대본:

아직 개선해야 할 점도 있습니다.

첫 번째로, 현재는 사용자가 실제로 말한 내용을 앱이 인식하지는 못합니다. 그래서 앞으로는 음성 인식 기능을 추가해서 사용자가 말한 내용을 자동으로 텍스트로 바꾸는 기능을 넣고 싶습니다.

두 번째로, AI 피드백 기능을 추가할 수 있습니다. 사용자가 설명한 내용을 바탕으로 부족한 부분이나 더 공부해야 할 내용을 알려주면 더 좋은 학습 도구가 될 수 있습니다.

세 번째로, 현재 기록은 저장만 가능하고 수정이나 삭제 기능은 부족합니다. 기록을 관리하기 쉽게 수정, 삭제 기능을 추가하면 더 완성도 있는 앱이 될 것 같습니다.

네 번째로, 현재는 브라우저의 `localStorage`에 기록을 저장합니다. 이 방식은 간단하지만, 다른 기기에서는 기록을 볼 수 없습니다. 그래서 로그인 기능과 서버 저장 기능을 추가하면 사용자별 기록 관리가 가능해질 것 같습니다.

마지막으로, 학습 통계 기능도 추가할 수 있습니다. 예를 들어 총 학습 시간, 가장 많이 공부한 주제, 날짜별 학습 횟수 등을 보여주면 사용자가 자신의 학습 습관을 확인하는 데 도움이 될 것 같습니다.
