# SpeakLog 발표 흐름

## 발표 핵심 한 줄

SpeakLog는 배운 내용을 직접 말로 설명해보고, 설명이 끝난 뒤 핵심 내용을 요약해서 기록으로 남기는 React 기반 학습 웹앱입니다.

## 발표 전에 켜둘 화면

- 시작 화면
- 학습 설정 화면
- 설명 진행 화면
- 요약 작성 화면
- 학습 기록 목록 / 상세 화면

발표할 때는 코드를 오래 보여주기보다, 화면을 넘기면서 "이 화면에서 어떤 React 기능을 썼는지"를 짧게 연결해서 말하면 됩니다.

## 5분 이내 버전

### 0:00-0:30 인사 + 프로젝트 소개

안녕하세요. 제가 만든 프로젝트는 **SpeakLog**입니다.

SpeakLog는 사용자가 오늘 배운 내용을 직접 말로 설명해보고, 설명이 끝난 뒤 핵심 내용을 요약해서 학습 기록으로 저장하는 웹앱입니다.

이 프로젝트를 만든 이유는 공부한 내용을 그냥 읽고 끝내는 것보다, 직접 설명해보면 내가 어느 부분을 이해했고 어느 부분이 부족한지 더 잘 알 수 있기 때문입니다.

### 0:30-1:00 전체 흐름 설명

사용 흐름은 간단합니다.

1. 학습 시작 버튼을 누릅니다.
2. 오늘 설명할 주제를 입력합니다.
3. 설명을 들어줄 청중을 선택합니다.
4. 화면을 보면서 직접 설명합니다.
5. 설명이 끝나면 핵심 내용을 요약합니다.
6. 기록을 저장하고 나중에 다시 확인합니다.

### 1:00-2:30 화면 시연

먼저 시작 화면에서는 최근 학습 기록을 확인할 수 있고, 새 학습을 시작할 수 있습니다.

학습 설정 화면에서는 주제를 입력하고 청중을 선택합니다. 예를 들어 "React 상태 관리"라고 입력한 뒤 청중을 선택하고 설명을 시작합니다.

설명 진행 화면에서는 선택한 청중 영상과 주제, 타이머가 보입니다. 사용자는 이 화면을 보면서 실제로 발표하듯이 설명하면 됩니다.

설명을 끝내면 요약 작성 화면으로 이동합니다. 여기서 방금 설명한 내용을 짧게 정리해서 저장합니다.

저장된 기록은 학습 기록 화면에서 다시 볼 수 있고, 상세 화면에서는 주제, 날짜, 설명 시간, 선택한 청중, 요약 내용을 확인할 수 있습니다.

### 2:30-4:15 코드 설명: useState, useEffect, localStorage

이 프로젝트에서 가장 중요한 파일은 `src/App.js`입니다. 화면 전환과 주요 데이터를 대부분 여기서 관리합니다.

첫 번째로 `useState`를 사용했습니다.

```js
const [screen, setScreen] = useState('home');
const [topic, setTopic] = useState('');
const [elapsed, setElapsed] = useState(0);
const [records, setRecords] = useState(() => {
  return JSON.parse(localStorage.getItem(STORAGE_KEY)) ?? [];
});
```

`screen`은 현재 어떤 화면을 보여줄지 정하는 값입니다. 예를 들어 `home`, `setup`, `session`, `summary`, `records`, `detail` 같은 값에 따라 다른 화면을 렌더링합니다.

`topic`은 사용자가 입력한 학습 주제, `elapsed`는 설명 시간, `records`는 저장된 학습 기록 목록입니다. 값이 바뀌면 React가 화면을 다시 그려주기 때문에 입력값, 타이머, 기록 목록이 바로 반영됩니다.

두 번째로 `useEffect`를 사용했습니다.

```js
useEffect(() => {
  if (screen !== 'session') {
    return undefined;
  }

  const timerId = window.setInterval(() => {
    setElapsed((value) => value + 1);
  }, 1000);

  return () => window.clearInterval(timerId);
}, [screen]);
```

설명 진행 화면에 들어가면 1초마다 `elapsed` 값이 증가해서 타이머가 작동합니다. 그리고 다른 화면으로 나가면 `clearInterval`로 타이머를 정리해서 계속 시간이 증가하지 않도록 했습니다.

세 번째로 `localStorage`를 사용했습니다.

```js
useEffect(() => {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(records));
}, [records]);
```

기록이 바뀔 때마다 `localStorage`에 저장합니다. 그래서 새로고침을 해도 학습 기록이 사라지지 않습니다.

### 4:15-5:00 마무리 + 개선점

정리하면 SpeakLog는 학습 주제를 입력하고, 말로 설명하고, 요약해서 기록으로 남기는 학습 보조 웹앱입니다.

React의 `useState`로 화면과 데이터를 관리했고, `useEffect`로 타이머와 저장 기능을 처리했으며, `localStorage`로 새로고침 후에도 기록이 유지되도록 만들었습니다.

앞으로는 실제 음성 인식 기능을 추가해서 사용자가 말한 내용을 자동으로 텍스트로 바꾸거나, AI 피드백 기능을 넣어서 설명에서 부족한 부분을 알려주는 방향으로 발전시킬 수 있을 것 같습니다.

이상으로 발표를 마치겠습니다. 감사합니다.

### 4:30-7:30 코드 설명

코드 설명은 `src/App.js`를 중심으로 하겠습니다. 이 파일에서 화면 전환, 입력값, 타이머, 기록 저장을 관리합니다.

#### 1. useState

먼저 `useState`입니다.

```js
const [screen, setScreen] = useState('home');
const [selectedAvatar, setSelectedAvatar] = useState(0);
const [topic, setTopic] = useState('');
const [currentTopic, setCurrentTopic] = useState('');
const [elapsed, setElapsed] = useState(0);
const [summary, setSummary] = useState('');
const [records, setRecords] = useState(() => {
  return JSON.parse(localStorage.getItem(STORAGE_KEY)) ?? [];
});
```

`screen`은 현재 어떤 화면을 보여줄지 관리합니다. 예를 들어 값이 `home`이면 시작 화면을 보여주고, `session`이면 설명 진행 화면을 보여줍니다.

이 방식 덕분에 별도의 페이지 이동 없이도 상태값만 바꿔서 화면을 전환할 수 있습니다.

`topic`은 설정 화면에서 입력하는 주제이고, `currentTopic`은 실제 설명을 시작할 때 확정된 주제입니다. 이렇게 나눈 이유는 입력 중인 값과 실제 세션에서 사용하는 값을 구분하기 위해서입니다.

`elapsed`는 설명 시간입니다. 설명 화면에 들어가면 1초씩 증가하고, 화면에는 `00:00` 형태로 표시됩니다.

`summary`는 사용자가 작성한 핵심 요약 내용이고, `records`는 지금까지 저장된 학습 기록 목록입니다.

즉 `useState`는 이 앱에서 화면 전환, 입력값, 타이머, 저장 기록처럼 계속 바뀌는 데이터를 관리하는 역할을 합니다.

#### 2. useEffect

다음은 `useEffect`입니다.

첫 번째 사용 위치는 기록 저장입니다.

```js
useEffect(() => {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(records));
}, [records]);
```

`records`가 바뀔 때마다 브라우저의 `localStorage`에 저장합니다. React 상태는 새로고침하면 사라지기 때문에, 기록을 유지하려면 따로 저장 공간이 필요합니다.

두 번째 사용 위치는 타이머입니다.

```js
useEffect(() => {
  if (screen !== 'session') {
    return undefined;
  }

  const timerId = window.setInterval(() => {
    setElapsed((value) => value + 1);
  }, 1000);

  return () => window.clearInterval(timerId);
}, [screen]);
```

`screen` 값이 `session`일 때만 타이머가 실행됩니다. 즉 설명 진행 화면에 들어갔을 때만 시간이 증가합니다.

그리고 `return` 부분에서 `clearInterval`을 실행합니다. 이 부분이 없으면 화면을 벗어난 뒤에도 타이머가 계속 돌아갈 수 있기 때문에, 화면이 바뀔 때 정리해주는 역할을 합니다.

설정 화면의 `SetupScreen.js`에서도 `useEffect`를 사용했습니다.

```js
useEffect(() => {
  topicInputRef.current?.focus();
}, []);
```

이 코드는 설정 화면이 처음 나타났을 때 주제 입력칸에 자동으로 포커스가 가게 합니다. 사용자가 바로 입력할 수 있게 하는 편의 기능입니다.

#### 3. localStorage

마지막으로 `localStorage`입니다.

처음 앱이 실행될 때는 저장된 기록을 읽어옵니다.

```js
const [records, setRecords] = useState(() => {
  try {
    return JSON.parse(localStorage.getItem(STORAGE_KEY)) ?? [];
  } catch {
    return [];
  }
});
```

브라우저에 저장된 기록이 있으면 가져오고, 없으면 빈 배열로 시작합니다. `try-catch`를 둔 이유는 저장된 값이 잘못된 형식일 때 앱이 멈추지 않도록 하기 위해서입니다.

그리고 기록이 추가되면 `useEffect`에서 다시 `localStorage`에 저장합니다.

```js
localStorage.setItem(STORAGE_KEY, JSON.stringify(records));
```

객체나 배열은 그대로 저장할 수 없기 때문에 `JSON.stringify`로 문자열로 바꿔 저장하고, 다시 가져올 때는 `JSON.parse`로 배열 형태로 바꿉니다.

이 덕분에 사용자가 새로고침을 해도 기록이 유지됩니다. 다만 `localStorage`는 같은 브라우저 안에서만 유지되기 때문에, 다른 기기에서도 기록을 보려면 나중에 서버 저장이나 로그인 기능이 필요합니다.

### 9:20-10:00 개선점 + 마무리

세 번째는 서버 저장 기능입니다. 현재는 `localStorage`에 저장하기 때문에 같은 브라우저에서만 기록을 볼 수 있습니다. 로그인과 서버 저장을 추가하면 여러 기기에서 기록을 이어서 볼 수 있을 것 같습니다.

React의 `useState`, `useEffect`, `localStorage`를 사용해서 화면 전환, 타이머, 기록 저장 기능을 구현했습니다.

## 코드 안 보여주는 화면 설명 버전

이 버전은 발표 중에 `App.js` 같은 코드 화면을 열지 않고, 실제 앱 화면만 보면서 설명하는 흐름입니다.

핵심은 세 가지입니다.

- `useState`: 화면에 보이는 값이 바뀌는 부분
- `useEffect`: 화면이 바뀌었을 때 자동으로 실행되는 부분
- `localStorage`: 새로고침해도 기록이 남는 부분

말할 때는 "코드에서는"보다 "이 화면에서는"으로 시작하면 질문이 덜 깊게 들어올 가능성이 높습니다.

### 화면 기준 기술 설명: 어떻게 썼고 왜 썼는지

#### useState

`useState`는 화면에서 계속 바뀌는 값을 관리하기 위해 사용했습니다.

어떻게 썼는지:

- 현재 어떤 화면인지 저장해서 시작 화면, 설정 화면, 설명 화면, 요약 화면, 기록 화면을 전환했습니다.
- 사용자가 입력한 학습 주제 값을 저장했습니다.
- 사용자가 선택한 청중 값을 저장했습니다.
- 설명 화면에서 증가하는 타이머 시간을 저장했습니다.
- 요약 화면에서 입력한 핵심 정리 내용을 저장했습니다.
- 저장된 학습 기록 목록을 관리했습니다.

왜 썼는지:

> 이 앱은 사용자가 버튼을 누르거나 값을 입력할 때 화면이 바로 바뀌어야 합니다. 일반 변수로 관리하면 값이 바뀌어도 화면이 자동으로 다시 그려지지 않기 때문에, React가 변화를 감지할 수 있도록 `useState`를 사용했습니다.

발표 멘트:

> `useState`는 화면에서 바뀌는 값들을 관리하는 데 사용했습니다. 예를 들어 현재 화면, 입력한 주제, 선택한 청중, 타이머 시간, 요약 내용, 저장된 기록 목록이 모두 사용자의 행동에 따라 바뀌기 때문에 상태로 관리했습니다. 상태가 바뀌면 React가 화면을 다시 그려주기 때문에 입력이나 버튼 클릭이 바로 화면에 반영됩니다.

#### useEffect

`useEffect`는 화면이 바뀌었을 때 자동으로 실행되어야 하는 기능을 처리하기 위해 사용했습니다.

어떻게 썼는지:

- 설명 진행 화면에 들어가면 타이머가 자동으로 시작되게 했습니다.
- 설명 화면을 벗어나면 타이머가 멈추도록 정리했습니다.
- 학습 기록이 바뀔 때마다 저장소에 자동 저장되게 했습니다.
- 설정 화면에 들어갔을 때 주제 입력칸에 바로 입력할 수 있도록 포커스를 주었습니다.

왜 썼는지:

> 타이머 시작, 저장소 저장, 입력칸 포커스처럼 화면을 그리는 것과 별개로 추가 실행이 필요한 기능들이 있습니다. 이런 기능들은 특정 화면에 들어가거나 특정 값이 바뀌었을 때 실행되어야 해서 `useEffect`를 사용했습니다.

발표 멘트:

> `useEffect`는 화면 변화에 맞춰 자동으로 실행되어야 하는 기능에 사용했습니다. 대표적으로 설명 화면에 들어가면 타이머가 시작되고, 다른 화면으로 이동하면 타이머가 멈추도록 했습니다. 또 기록이 추가되면 자동으로 저장되게 해서 사용자가 따로 저장 과정을 신경 쓰지 않아도 되도록 했습니다.

#### localStorage

`localStorage`는 학습 기록을 브라우저에 저장하기 위해 사용했습니다.

어떻게 썼는지:

- 앱이 처음 실행될 때 브라우저에 저장된 학습 기록을 불러왔습니다.
- 사용자가 요약을 작성하고 기록을 저장하면 기록 목록에 추가했습니다.
- 기록 목록이 바뀔 때마다 `localStorage`에 다시 저장했습니다.
- 새로고침 후에도 이전 기록이 화면에 다시 나타나게 했습니다.

왜 썼는지:

> React 상태만 사용하면 새로고침할 때 데이터가 사라질 수 있습니다. 학습 기록은 나중에 다시 확인해야 하는 데이터이기 때문에 브라우저에 남겨둘 필요가 있었습니다. 서버를 만들지 않고도 간단히 저장할 수 있어서 `localStorage`를 사용했습니다.

발표 멘트:

> `localStorage`는 저장된 학습 기록이 새로고침 후에도 유지되도록 사용했습니다. React 상태만 사용하면 새로고침할 때 기록이 사라질 수 있기 때문에, 기록 목록을 브라우저 저장소에 저장했습니다. 그래서 사용자가 앱을 다시 열어도 이전 학습 기록을 확인할 수 있습니다.

### 화면만 보면서 말하는 기술 설명 대본

아래 대본은 코드 화면을 열지 않고, 앱 화면만 띄워둔 상태에서 그대로 말하는 버전입니다.

#### 짧은 버전

이제 이 프로젝트에서 사용한 주요 기술을 화면 기준으로 설명드리겠습니다.

먼저 `useState`입니다. 이 앱에서는 사용자가 버튼을 누르거나 값을 입력할 때 화면이 계속 바뀝니다. 예를 들어 시작 화면에서 학습 시작 버튼을 누르면 설정 화면으로 이동하고, 설정 화면에서는 사용자가 입력한 주제와 선택한 청중이 화면에 반영됩니다. 또 설명 화면에서는 타이머 시간이 계속 바뀌고, 요약 화면에서는 사용자가 작성하는 요약 내용이 바뀝니다.

이렇게 화면에서 계속 변하는 값들은 `useState`로 관리했습니다. 일반 변수로 값을 저장하면 값이 바뀌어도 React 화면이 자동으로 다시 그려지지 않기 때문에, React가 변화를 알고 화면에 반영할 수 있도록 `useState`를 사용했습니다.

다음은 `useEffect`입니다. `useEffect`는 특정 화면에 들어갔을 때 자동으로 실행되어야 하는 기능에 사용했습니다. 대표적으로 설명 진행 화면의 타이머가 있습니다. 사용자가 설명 화면에 들어가면 타이머가 자동으로 시작되고, 설명을 끝내서 다른 화면으로 이동하면 타이머가 멈춰야 합니다.

이런 기능은 단순히 화면을 보여주는 것과는 별도로, 화면 상태가 바뀌었을 때 실행되어야 하기 때문에 `useEffect`를 사용했습니다. 그래서 설명 화면에서는 1초마다 시간이 증가하고, 화면을 벗어나면 타이머가 정리되도록 만들었습니다.

마지막은 `localStorage`입니다. 사용자가 요약을 작성하고 기록을 저장하면, 그 기록은 나중에 다시 확인할 수 있어야 합니다. 그런데 React의 상태만 사용하면 새로고침했을 때 기록이 사라질 수 있습니다.

그래서 학습 기록은 브라우저 저장소인 `localStorage`에 저장했습니다. 덕분에 사용자가 페이지를 새로고침하거나 앱을 다시 열어도 이전에 저장한 학습 기록을 기록 화면에서 다시 확인할 수 있습니다.

정리하면, `useState`는 화면에서 바뀌는 값들을 관리하기 위해 사용했고, `useEffect`는 타이머처럼 화면 변화에 따라 자동으로 실행되는 기능을 처리하기 위해 사용했습니다. 그리고 `localStorage`는 학습 기록이 새로고침 후에도 유지되도록 하기 위해 사용했습니다.

#### 조금 더 자세한 버전

이 프로젝트에서 사용한 React 기능은 화면 흐름에 맞춰 설명드리겠습니다.

먼저 시작 화면과 설정 화면에서는 `useState`를 많이 사용했습니다. 현재 어떤 화면을 보여줄지, 사용자가 입력한 학습 주제가 무엇인지, 어떤 청중을 선택했는지 같은 값들이 모두 사용자의 행동에 따라 바뀝니다. 예를 들어 학습 시작 버튼을 누르면 설정 화면으로 바뀌고, 청중 카드를 누르면 선택된 청중이 바뀝니다.

이런 값들은 화면에 바로 반영되어야 하기 때문에 `useState`로 관리했습니다. `useState`를 사용하면 값이 바뀔 때 React가 화면을 다시 렌더링해줍니다. 그래서 사용자가 주제를 입력하거나 청중을 선택했을 때 화면이 바로 바뀌는 것입니다.

설명 진행 화면에서는 `useEffect`를 사용했습니다. 이 화면에 들어오면 타이머가 자동으로 시작되고, 1초마다 시간이 증가합니다. 이 타이머는 사용자가 설명하는 동안에만 작동해야 하고, 설명을 끝내서 요약 화면으로 이동하면 멈춰야 합니다.

그래서 `useEffect`를 사용해서 설명 화면에 들어왔을 때 타이머를 시작하고, 화면을 벗어날 때 타이머를 정리하도록 했습니다. 이 방식을 사용한 이유는 타이머가 화면 표시와 별개로 계속 실행되는 기능이기 때문에, 필요한 화면에서만 실행되도록 관리해야 했기 때문입니다.

요약 작성 화면에서도 `useState`를 사용했습니다. 사용자가 요약 내용을 입력하면 그 입력값이 계속 바뀌기 때문에 상태로 저장했습니다. 그리고 작성 완료 버튼을 누르면 주제, 청중, 설명 시간, 요약 내용을 하나의 학습 기록으로 만들었습니다.

마지막으로 기록 화면에서는 `localStorage`를 사용했습니다. 학습 기록은 한 번 보고 끝나는 데이터가 아니라 나중에 다시 복습하기 위해 남겨야 하는 데이터입니다. 그런데 React 상태만 사용하면 새로고침했을 때 기록이 사라질 수 있습니다.

그래서 기록 목록을 `localStorage`에 저장했습니다. 앱이 처음 실행될 때는 저장되어 있던 기록을 불러오고, 새 기록이 추가되면 다시 저장되게 했습니다. 이 덕분에 새로고침을 해도 기록이 유지됩니다.

즉 이 프로젝트에서 `useState`는 사용자의 행동에 따라 바뀌는 화면 값을 관리하기 위해, `useEffect`는 타이머나 자동 저장처럼 특정 상황에서 실행되어야 하는 기능을 처리하기 위해, `localStorage`는 학습 기록을 브라우저에 남기기 위해 사용했습니다.

## 화면 설명 10분 이내 버전

### 0:00-0:50 인사 + 프로젝트 소개

안녕하세요. 제가 발표할 프로젝트는 **SpeakLog**입니다.

SpeakLog는 사용자가 배운 내용을 직접 말로 설명하고, 설명이 끝난 뒤 핵심 내용을 요약해서 기록으로 남기는 학습 웹앱입니다.

공부한 내용을 그냥 읽고 끝내면 이해한 것처럼 느껴질 수 있지만, 직접 설명해보면 어떤 부분을 정확히 알고 있고 어떤 부분이 부족한지 더 잘 알 수 있습니다.

그래서 이 프로젝트는 "말로 설명하는 학습 과정"을 웹앱으로 만든 것입니다.

### 0:50-1:40 전체 사용 흐름

전체 흐름은 다음과 같습니다.

1. 학습 시작 버튼을 누릅니다.
2. 오늘 설명할 주제를 입력합니다.
3. 설명을 들어줄 청중을 선택합니다.
4. 설명 화면에서 직접 말로 설명합니다.
5. 설명이 끝나면 핵심 내용을 요약합니다.
6. 기록을 저장하고 나중에 다시 확인합니다.

이 앱은 기능을 많이 넣기보다, 학습 과정이 자연스럽게 이어지는 것에 집중했습니다.

### 1:40-2:40 시작 화면 설명

먼저 시작 화면입니다.

시작 화면에는 앱의 소개 문구와 학습 시작 버튼이 있습니다. 그리고 아래쪽에는 최근 학습 기록이 표시됩니다.

최근 기록을 시작 화면에 배치한 이유는 사용자가 앱에 들어왔을 때 바로 이전 학습 내용을 확인할 수 있게 하기 위해서입니다.

이 화면에서 최근 기록은 저장된 기록 목록에서 최신 3개만 보여주는 방식입니다. 기록이 없을 때는 아직 기록이 없다는 안내가 나타납니다.

이처럼 기록 목록은 저장된 데이터에 따라 화면이 달라지기 때문에 React의 `useState`로 관리했습니다.

### 2:40-3:50 학습 설정 화면 설명

학습 시작 버튼을 누르면 학습 설정 화면으로 이동합니다.

이 화면에서는 두 가지를 정합니다. 첫 번째는 오늘 설명할 학습 주제이고, 두 번째는 설명을 들어줄 청중입니다.

주제 입력칸에는 사용자가 직접 내용을 입력합니다. 사용자가 입력하는 값은 계속 바뀌기 때문에 `useState`로 관리했습니다.

청중 선택도 마찬가지입니다. 청중 카드를 클릭하면 선택된 청중이 바뀌고, 화면에서는 선택된 카드가 표시됩니다.

그리고 설정 화면에 들어왔을 때 바로 주제를 입력할 수 있도록 입력칸에 자동으로 포커스가 가게 했습니다. 이런 "화면이 처음 나타났을 때 한 번 실행되는 기능"에는 `useEffect`를 사용했습니다.

### 3:50-5:20 설명 진행 화면 설명

설명 시작 버튼을 누르면 설명 진행 화면으로 이동합니다.

이 화면에서는 상단에 학습 주제와 타이머가 표시되고, 가운데에는 선택한 청중 영상이 나옵니다.

사용자는 이 화면을 보면서 실제로 누군가에게 설명하듯이 말하면 됩니다.

여기서 중요한 기능은 타이머입니다. 타이머는 사용자가 설명 화면에 들어왔을 때 자동으로 시작되어야 하고, 설명 화면을 벗어나면 멈춰야 합니다.

그래서 이 부분에는 `useEffect`를 사용했습니다.

`useEffect`는 특정 화면에 들어왔을 때 자동으로 실행해야 하는 작업에 적합합니다. 이 프로젝트에서는 설명 화면에 들어왔을 때 1초마다 시간이 증가하도록 만들었습니다.

또 설명 완료 버튼을 눌러 다른 화면으로 이동하면 타이머가 더 이상 실행되지 않도록 정리했습니다.

즉 설명 진행 화면에서는 `useEffect`가 타이머를 시작하고 정리하는 역할을 합니다.

### 5:20-6:30 요약 작성 화면 설명

설명을 완료하면 요약 작성 화면으로 이동합니다.

요약을 직접 작성하게 한 이유는, 말로 설명한 내용을 다시 한 번 정리하면서 학습 내용을 더 확실히 기억하게 하기 위해서입니다.

요약 입력값도 사용자가 작성하면서 계속 바뀌는 값이므로 `useState`로 관리했습니다.

작성 완료 버튼을 누르면 지금까지의 정보가 하나의 기록으로 저장됩니다. 저장되는 내용은 날짜, 주제, 선택한 청중, 설명 시간, 요약 내용입니다.

다만 `localStorage`는 현재 사용하는 브라우저 안에만 저장된다는 한계가 있습니다. 그래서 나중에 더 발전시킨다면 로그인 기능이나 서버 저장 기능을 추가할 수 있습니다.

## 세 가지 코드 포인트만 빠르게 말해야 할 때

### useState

`useState`는 바뀌는 값을 관리하기 위해 사용했습니다.

- `screen`: 현재 화면 관리
- `topic`: 사용자가 입력한 주제
- `selectedAvatar`: 선택한 청중
- `elapsed`: 설명 시간
- `summary`: 요약 내용
- `records`: 저장된 학습 기록

말하기:

> 이 앱에서는 화면 전환과 입력값, 타이머, 기록 목록처럼 계속 바뀌는 데이터가 많아서 `useState`로 관리했습니다. 상태가 바뀌면 React가 자동으로 화면을 다시 그려주기 때문에 사용자가 입력하거나 버튼을 눌렀을 때 화면이 자연스럽게 바뀝니다.

### useEffect

`useEffect`는 특정 상태가 바뀌었을 때 실행해야 하는 작업에 사용했습니다.

- `records`가 바뀌면 `localStorage`에 저장
- `screen`이 `session`이면 타이머 시작
- 화면을 벗어나면 `clearInterval`로 타이머 정리
- 설정 화면 진입 시 주제 입력칸 자동 포커스

말하기:

> `useEffect`는 화면이 바뀌거나 기록이 바뀌었을 때 추가로 실행해야 하는 기능에 사용했습니다. 특히 설명 화면에서는 타이머를 시작하고, 다른 화면으로 이동하면 타이머를 정리하도록 만들었습니다.

### localStorage

`localStorage`는 학습 기록을 브라우저에 저장하기 위해 사용했습니다.

- 앱 시작 시 저장된 기록 읽기
- 기록 추가 시 다시 저장
- 새로고침 후에도 기록 유지

말하기:

> React state만 사용하면 새로고침할 때 기록이 사라지기 때문에 `localStorage`를 사용했습니다. 기록 배열을 `JSON.stringify`로 문자열로 바꿔 저장하고, 다시 불러올 때는 `JSON.parse`로 배열로 바꿉니다.

## 예상 질문 답변

### Q. 왜 `localStorage`를 사용했나요?

서버 없이도 브라우저 안에 데이터를 저장할 수 있어서 사용했습니다. 학습 기록처럼 간단한 데이터는 `localStorage`로 저장하면 새로고침 후에도 유지할 수 있습니다.

### Q. `useEffect`에서 왜 `clearInterval`을 하나요?

설명 화면을 벗어난 뒤에도 타이머가 계속 실행되는 문제를 막기 위해서입니다. 화면이 바뀌면 기존 타이머를 정리해서 불필요한 실행을 방지합니다.

### Q. `useState` 대신 일반 변수를 쓰면 안 되나요?

일반 변수는 값이 바뀌어도 React 화면이 다시 렌더링되지 않습니다. 사용자 입력이나 타이머처럼 화면에 바로 반영되어야 하는 값은 `useState`로 관리해야 합니다.

### Q. 아쉬운 점은 무엇인가요?

현재는 실제 음성을 자동으로 인식하지는 않고, 사용자가 직접 요약을 입력해야 합니다. 나중에는 음성 인식과 AI 피드백, 서버 저장 기능을 추가하고 싶습니다.

### Q. 이 프로젝트에서 가장 중요한 기능은 무엇인가요?

말로 설명하는 화면과 기록 저장 기능입니다. 학습한 내용을 직접 설명하고, 그 결과를 다시 볼 수 있게 만든 것이 핵심입니다.
